Agentyc's Architecture

The Building Blocks

Agents

Agents are entities that are assigned a role and specialization by the user in plain English.

Models

Models are the brains behind Agents.

Skills

Skills are Python functions that enable Agents to search & browse the web or use third-party APIs.

Missions

Missions are tasks with well-defined objectives that are assigned to a group of Agents.

Agentyc's Data Model

The foundation of any application is the data model. The information architecture, UI & API designs are all based on this foundation.

I applied principles of object-oriented design to identify the building blocks, their individual attributes and their relationship with each other to define Agentyc's data model.

The objects, attributes and relationships that form Agentyc's data model

App Architecture

  • Agent, Mission, Model API keys and Skill data is stored in a SQLite database
  • When the user starts a Mission, the Agentyc app dynamically generates a Python script with the Mission configuration and a websocket server
  • The script is deployed inside Agentyc's Docker container which is built on Python 3.11 Slim + Microsoft AutoGen + other dependencies
  • The script connects to Models in the cloud or Models running locally on the macOS device through Ollama or LMStudio
  • The script communicates with the SwiftUI app through websockets to send real-time updates and stream responses
